VScode基本配置

1. 主题

1.1 颜色主题

配置路径：左下角齿轮 ---> 颜色主题

推荐颜色主题

深色主题 软件内置 深色+（Visual Studio）
浅色主题 扩展商城 Github Theme（light）

1.2 文件图标主题

配置路径：左下角齿轮 ---> 文件图标主题

推荐文件图标主题

扩展商城---VScode Great Icons

2. 与Matlab

2.1 扩展安装

安装路径：扩展商城 ---> Matlab 和 MatlabSnippets以及 Code Runner

2.2 修改setting.json

修改路径：左下角齿轮 ---> 设置 ---> 右上角，打开设置(json) ---> 添加如下代码


    //Matlab路径
    "matlab.matlabpath": "D:\\Matlab\\bin\\matlab.exe", 
    //lint工具路径
    "matlab.mlintpath": "D:\\Matlab\\bin\\win64\\mlint.exe",
    //提示编码
    "matlab.linterEncoding": "gb2312",
    "files.associations": {"*.m":"matlab"},
    //m文件的编码应该为英文的
    "[matlab]": {
    "files.encoding": "gb2312"
    },

    //Matlab运行配置(code runner插件的设置),如不采用2.3.1的方法则可删去
    "code-runner.executorMap": {
    "matlab": "cd $dir && matlab -nosplash -nodesktop -r $fileNameWithoutExt"
    }

2.3 脚本运行

2.3.1 Code Runner

对于在VScode中已经编写好的.m文件，右键点击Run Code或使用快捷键Ctrl+Alt+N使之运行，如代码无误，运行成功后会有Matlab的终端窗口弹出，以此显示运行结果。

2.3.2 代码编辑

配置路径：Matlab主页 ---> 预设 ---> 编辑器/调试器[1] ---> 文本编辑器，添加路径即可

  1. 上述配置路径不是必需的，可以在VScode中编辑好Matlab代码保存后，在Matlab中以其转到Matlab内置的编辑/调试器中进行调试。
  2. 经过了上述配置操作，点击Matlab资源管理器里的代码文件，则会立即跳转到VScode中打开，之后Matlab所有编辑和调试操作都在VScode中进行。
